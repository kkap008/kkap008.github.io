Form

Form이란?

"form"요소는 정보를 제출하기 위한 대화형 컨트롤 을 포함하는 문서 구획을 나타냅니다.

"form"의 속성들

  • accept-cahrset
    폼 데이터(form data)가 서버로 제출될 때 사용되는 문자 인코딩(character encoding) 방식을 명시함.
  • action
    폼 데이터가 서버로 제출될 때 해당 데이터가 도착할 URL을 명시함.
  • autocomplete
    "form" 요소에서 자동 완성 기능을 사용할지 여부를 명시함.
  • enctype
    폼 데이터가 서버로 제출될 때 해당 데이터가 인코딩되는 방법을 명시함.
    (단, "form" 요소의 method 속성값이 “post”인 경우에만 사용할 수 있음)
  • method

    폼 데이터가 서버로 제출될 때 사용되는 HTTP 메소드를 명시함.

    HTTP 메소드
    GET
    • 이름/값 쌍의 URL에 양식 데이터를 추가합니다.
    • 중요한 데이터를 보낼 때 GET를 사용하지 마십시오!
      (URL에 표시됩니다.)
    • URL의 길이는 약 3,000자로 제한됩니다.
    • 결과를 책갈피로 지정할 양식 제출에 유용합니다.
    • GET는 Google의 쿼리 문자열과 같은 비보안 데이터에 더 좋습니다.
    • HTTP 요청은 브라우저에 의해 캐시되어(cached) 저장됩니다.
    POST
    • HTTP 요청 본문 내부에 양식 데이터를 추가합니다.
      (URL에는 데이터가 표시되지 않음)
    • 크기 제한이 없습니다.
    • POST가 있는 양식 제출을 책갈피로 지정할 수 없습니다.
    • 브라우저에 의해 캐시되지 않으므로, 브라우저 히스토리에도 남지 않습니다.
  • name
    "form" 요소의 이름을 명시함.
    name 속성은 서버로 제출된 폼 데이터(form data)를 참조하기 위해 사용되거나,
    자바스크립트에서 요소를 참조하기 위해 사용됩니다.
  • novalidata
    폼 데이터(form data)를 서버로 제출할 때 해당 데이터의 유효성을 검사하지 않음을 명시합니다.
    "이메일 주소에 '@'를 포함해 주세요...." 이런 애기 안하고 바로 데이터 전송
  • rel
    연결된 리소스와 현재 문서 간의 관계를 지정합니다.
  • target
    폼 데이터를 서버로 제출한 후 받는 응답이 열릴 위치를 명시함.

"form"의 구성 요소들

  • 구획

    "fieldset" 요소
    여러 대화형 컨트롤러와 여러 라벨을 각각 목적에 맞게 구획으로 나눠 분류할 때 사용한다.

  • 레이블링

    "label" 요소
    • 여러 폼 요소에 대한 레이블을 정의합니다.
    • 사용자가 입력 요소에 초점을 맞출 때 화면 판독기가 레이블을
      크게 읽기 때문에 "label" 요소는 화면 판독기 사용자에게 유용합니다.
    • 사용자가 "label" 요소 내의 텍스트를 클릭하면 라디오 버튼/확인란이 전환되기 때문에
      라디오 버튼 또는 확인란과 같은 매우 작은 영역(예: 라디오 버튼)을 클릭하는 데에도 도움이 됩니다.
    • "input" 태그와 그룹화 시킬려면 "label" 태그 "for" 속성에 "input" 태그 "ID" 작성
    • "form" 태그와 연결 시킬 수 있다.

  • 입력

    • 버튼 요소
      "type" 속성에 "button"을 명시해주어야한다.

      (※ <input>과 <button>은 기본적으로 "submit"으로 동작한다.)
      (★ <button>이 디자인 작업에 더 적합하다.)

    • 리스트 요소
      공통사항 : 자식요소가 "option" 태그이다.
      • select
      • datalist
    • 텍스트 요소

      • textarea

      • input : "type" 속성이 "hidden"이 아닌 경우

  • 출력 요소
    "output" 요소
    + = 60
