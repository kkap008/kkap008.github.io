멀티미디어 임베드 요소

"Video" 요소 설명

예시

cat1.mp4

관련 태그 및 속성들
<video> 태그
  • autoplay : 웹페이지 시작 시 자동재생
    MDN - 자동 재생 가이드 참조
  • contrlos : 비디오 컨트롤을 표시하도록 지정합니다(예: 재생/일시 중지 단추).
  • loop : 반복 재생
  • muted : 음소거
  • poster : 썸네일
    (※ 비디오 로드 시 / 재생 버튼 클릭 전에 보여진다. "autoplay" 속성 사용 시 로드 시간이 짧아져서 못볼 수 있다.)
  • preload
    auto : 페이지가 로드될 때 브라우저가 전체 비디오를 로드
    (비디오 파일도 같이 로드된다.)
    metadata : 페이지가 로드될 때 브라우저가 메타데이터만 로드
    (비디오 메타데이터만 로드)
    none : 페이지가 로드될 때 동영상을 로드하지 않음
    (비디오 파일을 로드하지 않음. 재생하기 전 비디오 시간 표시 안됨)
    ※ "autoplay" 속성 사용 시 "preload" 속성은 무시된다.
  • src : 비디오 경로
    (※ <source> 태그를 이용한 대체 영상을 제공 안할 시 사용. 즉 단일 영상만 제공 시 사용)
<source> 태그
  • src : 비디오 경로
  • type : MIME-type
    (※ MIME-type? 텍스트로만 가능했던 초기 인터넷 전자메일에,
    파일,이미지,오디오 등도 첨부 가능케 한 다목적용 확장 타입 규격)
    (구성? type/subtype)
    (ex text/css, vedio/mp4 ...)
<track> 태그

미디어 요소의 자식으로서, 자막 등 시간별 텍스트 트랙(시간 기반 데이터)를 지정할 때 사용

파일 종류
WebVTT(Web Video Text Tracks) - .vtt 파일
Timed Text Markup Language(TTML)
  • default : 영상의 사용할 기본 트랙을 설정
  • kind : 텍스트 트랙의 종류
    (※ 기본은 "subtitles")
    (※ 유효하지 않는 값은 "metadata"로 간주)
    • subtitles
      번역 자막
      ex) 소리는 영어 -- 자막은 한국어
    • captions
      • 청각이 안좋은 사람들을 위한 자막
      • 소리를 들을 수 없거나 켤 수 없는 경우에 사용
      • 번역 자막 + 비언어적 효과 자막 (효과음)
    • descriptions
      • 비디오 콘텐츠의 텍스트 설명
      • 비디오를 볼 수 없는 환경 / 시각이 안좋은 사용자에게 적합
      • 화면에 표시되지 않는다
    • chapters
      • 챕터 타이틀은 미디어 리소스를 탐색할 때 사용
      • 화면에 표시되지 않는다
    • metadata
      • 텍스트 트랙이 스크립트에 사용되는 콘텐츠임을 명시
      • 화면에 표시되지 않는다
  • src : 트랙 파일의 URL
  • label : 트랙의 제목을 명시
  • srclang
    • 트랙의 데이터 언어를 명시
      (단, kind 속성값이 “subtitles”인 경우에만 사용할 수 있음)
    • IETF(국제 인터넷 표준화 기구) 기술 사항 중 ISO 639-1 사용
사용 가능한 비디오 파일 종류
  • ★mp4 - 가장 많은 브라우저 지원
  • webm - 사파리 지원 애매함
  • ogg - 사파리 지원 애매함

"Audio" 요소 설명

예시
관련 태그 및 속성들
  • autoplay : 오디오 자동 실행
  • controls : 제어기가 표시됨
  • loop : 자동재생
  • muted : 페이지 로드 시 음소거 상태
  • preload (=비디오 태그 속성과 똑같다)
    • none
    • metadata
    • auto
  • src : 오디오 파일 위치 명시
    (※ <audio> 태그 와 함께 사용)
    (※ <source> 태그 와 함께 사용)
사용 가능한 오디오 파일 종류
★mp3
Moving Picture Experts Group에 의해 개발되었으며, MPEG-1의 오디오 규격으로 개발된 손실 압축형 파일 형식
wav
IBM과 Microsoft에 의해 개발되었으며, 개인용 PC에서 오디오를 재생하기 위한 IBM과 Microsoft의 표준 오디오 파일 형식
ogg
Xiph 재단에 의해 개발되었으며, MP3의 대안으로 개발된 특허권으로 보호되지 않는 개방형 공개 멀티미디어 파일 형식
